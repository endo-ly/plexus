# Muxport システムアーキテクチャ

## 目的

Muxport は tmux を中心に据えたランタイムをモバイルから扱うためのリポジトリである。
ターミナルアクセスを起点に、将来のワーカーオーケストレーションを載せられるランタイム基盤を提供する。

主要コンポーネントと責務境界を定義する。

## 全体構成

```text

│                         Mobile Client                        │
│  frontend/                                                   │

└──────────────────────────────┬───────────────────────────────┘
                               │ HTTP / WebSocket
                               ▼

│  - session list API                                          │
│  - snapshot API                                              │
│  - ws token issue                                            │

│  - push / webhook integration                                │

                               │
                               ▼

│                       Execution Runtime                      │
│  tmux sessions + host runtime                                │
│  - long-lived execution context                              │
│  - attach / reconnect point                                  │
│  - session-oriented runtime state                            │
└──────────────────────────────────────────────────────────────┘
```

## 主要コンポーネント

### フロントエンド

`frontend/` は Android 向けターミナルクライアント。Muxport ランタイムの利用面を担う。

- tmux セッション一覧表示
- ターミナルセッションへの接続
- ゲートウェイ接続設定の保持
- スナップショット / 再接続 / モバイル入力補助

フロントエンドはランタイムの利用者であり、tmux やワーカーライフサイクルを直接管理しない。

### ゲートウェイ

`gateway/` は tmux ランタイムを API 化する層。フロントエンドと実行環境の境界に位置する。

- セッション一覧 / 詳細の提供
- WebSocket ターミナル接続の受け口
- WebSocket トークン発行
- ターミナルスナップショットの取得
- プッシュ / webhook などランタイム連携

ゲートウェイはターミナルアクセスの入口であり、ランタイムポリシーと通信を担当する。

### tmux ランタイム

tmux セッションは Muxport が扱う実行の基本単位である。

- 実行継続の単位
- 再接続可能なエンドポイント
- ランタイム状態を観測する対象

ターミナル UI の背後にある実体として tmux セッションを扱う。

## 責務境界

| レイヤー | 責務 | 持たない責務 |
| --- | --- | --- |
| フロントエンド | モバイル UI、設定、ターミナル接続体験 | tmux 制御の本体、ランタイムポリシー |
| ゲートウェイ | ランタイム API、通信、接続仲介 | UI 状態管理、画面遷移 |
| tmux ランタイム | 実行継続、アタッチ対象、実行状態 | モバイル UX、API 表現 |

## 現在の焦点と将来の拡張

Issue 93 の移行フェーズでは、ターミナルゲートウェイとフロントエンドターミナルクライアントを独立リポジトリとして成立させることを優先する。

現時点の中核はターミナルアクセスであり、Muxport はターミナルアクセスとランタイム API の責務境界を明確にする設計になっている。
ターミナル層とランタイム API は密接に連携しつつも、責務を混在させない。
